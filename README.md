# Predictive Maintenance with Reinforcement Learning

This project implements a **Reinforcement Learning environment for predictive maintenance** based on a probabilistic failure model.
The environment simulates the degradation of industrial equipment and allows an RL agent to learn optimal maintenance policies.

The goal is to **balance operational reward with maintenance costs and failure risk**.

The environment follows the **Gymnasium API**.

---

# Project Overview

Industrial machines degrade over time due to usage.
Poor maintenance strategies can lead to: *unnecessary maintenance costs*, *unexpected failures* or *production downtime* 

This project models the problem as a **Partially Observed Markov Decision Process (POMDP)** where an agent must decide whether to: 

* 0 : continue operating
* 1 : inspect the machine
* 2 : perform maintenance

based on a state defined by machine sensor data, and partially observed latent quantity.

# Partially Observed Markov Decision Process

## State (Observation)

The state corresponds to machine sensor measurements and derived indicators.

```
[
Air temperature,
Process temperature,
Rotational speed,
Torque,
Tool wear,
Type_L,
Type_M,
temp_diff,
torque_speed_ratio,
wear_rate,
observed_risk,
uncertainty
]
```

Derived features:

* `temp_diff = Process temperature − Air temperature`
* `torque_speed_ratio = Torque / Rotational speed`
* `wear_rate = Tool wear / Rotational speed`

Additional variables:

* **observed_risk** → noisy estimation of failure probability
* **uncertainty** → uncertainty of the risk estimate

The environment assumes that the true failure risk of the machine is not directly observable by the agent. Instead, the agent receives a noisy estimate of this risk, denoted as observed_risk. This observation is obtained by perturbing the true risk with stochastic noise, which reflects the imperfect nature of predictive maintenance systems and sensor measurements in real industrial settings.

---
## Three possible actions 

### Continue

* increases machine wear
* increases operational reward
* increases failure probability

### Inspect

* small cost
* reveals accurate risk estimation

### Maintain

* resets wear
* reduces failure probability
* high cost

---

# Reward Structure

The reward function encourages **efficient maintenance strategies**.

## Positive rewards

```
+ survival_reward (1.5)
```

for continuing operation without failure.

---

## Costs

| Action   | Cost |
| -------- | ---- |
| Inspect  | -0.2 |
| Maintain | -0.5 |
| Failure  | -18  |

---

## Risk-based penalties

Additional penalties encourage safe operation:

* Continuing when risk is high → penalty
* Performing maintenance too early → penalty
* Not inspecting during uncertain medium-risk states → penalty

This reward shaping guides the agent toward safer and more informative strategies while still requiring it to reason under partial observability, since the agent only has access to a noisy estimate of the risk (observed_risk).

# Model-driven risk estimation

The failure probability used in the environment is not generated by a handcrafted rule.
Instead, it is produced by a trained machine learning model. (XGBoost trained in file *risk_model.py*)


```
risk_model.predict_proba(features)
```

A failure event occurs with probability:

```
P(failure) = predicted_risk
```

If failure occurs:

```
reward -= failure_cost
episode terminates
```

---

# Partial Observability

The true failure probability is **not perfectly observable**.

Instead, the agent observes:

```
observed_risk = true_risk + noise
```

Noise depends on the action:

| Situation     | Noise          |
| ------------- | -------------- |
| No inspection | high noise     |
| Inspection    | very low noise |

This creates a **POMDP-like setting** where the agent must decide when it is worth paying for better information.

---

# Episode Termination

An episode ends when:

* a **failure occurs**
* the **maximum number of steps (40)** is reached
